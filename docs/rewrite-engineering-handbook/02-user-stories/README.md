# User Story Documents

## Purpose

This directory converts product requirements into Agile epics, user stories, use cases, and story maps.

## Read First

1. [epics.md](./epics.md)
2. [user-stories.md](./user-stories.md)
3. [story-map.md](./story-map.md)
4. [use-case-diagram.md](./use-case-diagram.md)

## Files

| File | Purpose |
|---|---|
| [epics.md](./epics.md) | High-level product and engineering epics |
| [user-stories.md](./user-stories.md) | Role-based Agile user stories |
| [story-map.md](./story-map.md) | User activity map and release slicing |
| [use-case-diagram.md](./use-case-diagram.md) | Mermaid use-case diagrams |

## Update Rules

- Every P0 user-facing feature should have at least one user story.
- Story acceptance notes must stay aligned with [../01-product/acceptance-criteria.md](../01-product/acceptance-criteria.md).
- If a story changes system boundaries, update the technical design and backlog too.
