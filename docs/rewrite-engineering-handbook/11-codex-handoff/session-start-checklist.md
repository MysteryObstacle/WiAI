# Session Start Checklist

Before coding:

- Read document map.
- Confirm current sprint.
- Confirm target files.
- Run `git status --short`.
- Check for user changes.
- Do not overwrite unrelated changes.
- Start with tests where practical.

During coding:

- Keep changes scoped to the sprint.
- Update docs when behavior changes.
- Run package-level tests after each task.
- Do not introduce forbidden old names.

Before final response:

- Run relevant verification.
- Report commands run.
- Report files changed.
- State remaining gaps clearly.
